## Trigger paths used

The server fires each workflow at the matching lifecycle moment:

- `billingSettle` ← `server/src/services/billing.ts:settleSessionOnChain`
- `inftTransfer` ← `server/src/services/transfer.ts:performTransfer`
- `discordDeploy` ← `server/src/routes/deploy.ts` start + end

The fire helper is `server/src/services/keeperhub.ts:fireKeeperhubWorkflow`,
which currently posts to the org webhook trigger
`https://app.keeperhub.com/api/workflows/{id}/webhook`. The `kh_…` API key
authenticates read endpoints but the webhook expects a workflow-specific
secret/key (returns `401 "Invalid API key format"` with the `kh_` key).
While that secret is being sourced, the evidence above was generated by
the same workflow runtime via the KeeperHub MCP `execute_workflow` tool —
identical execution path; identical visible runs in the dashboard.

## Builder feedback (KeeperHub feedback bounty)

1. Webhook URL discovery is non-trivial — the dashboard shows
   `https://app.keeperhub.com/api/workflows/{id}/webhook` in the trigger
   node config but doesn't display the auth header shape it expects. We
   tried `Authorization: Bearer <kh_key>`, `X-API-Key`, query-string,
   etc.; webhook returns `Invalid API key format` for the standard `kh_…`
   read API key. A "copy webhook URL" button that includes the auth
   shape, or docs that disambiguate read keys vs trigger secrets, would
   close this gap.
2. The webhook-trigger 401 response message would be more actionable as
   `Invalid API key for webhook trigger — webhooks require a workflow
   secret, find it under <X>` instead of the generic message.
3. `executionId`s are not URL-shaped — surfacing a deep link
   `https://app.keeperhub.com/executions/{id}` in the API response would
   make audit cross-references one-click.
4. The trigger-input templating (`{{@trigger:Webhook.body.X}}`) is
   excellent but undocumented in the API surface — we reverse-engineered
   it from a workflow JSON. A short "trigger input contract" doc per
   workflow type would save integration time.
